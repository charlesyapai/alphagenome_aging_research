1. Overview: two models, one inference pipeline

This project connects a large pretrained model to a small domain classifier. The distinction is load-bearing:

AlphaGenome pretrained Aging classifier ours
Trained on Multi-omic tracks, ~700 biosamples, Nature 2026 AlphaGenome features on ~10k variants
Role here Frozen inference — scores each variant Decision layer — aging vs matched control
Architecture Large multi-output neural net Random Forest / logistic regression
Input 1-Mb DNA window around variant 115-d feature vector per variant
Output ~26k predictions × tissue × track P(aging) + trait category
What this means: the biology signal comes from AlphaGenome. The engineering contribution of this project is the inference pipeline: the design that connects a variant input to a defensible, honestly-validated prediction.

3. Matched controls — why MAF matching matters

For a classifier to have meaningful output, the aging and control distributions must differ only on the biology we care about. In v1, controls were chromosome-matched only. This let a simple bias into the data:

Aging (v1)Control (v1)
SourceGWAS — discovery biased toward common Ensembl — random — dominated by rare
Mean gnomAD AF0.31unknown (93%)
Effective signal "is the variant common?" — not "aging vs not"

v2.1 replaces the controls with one-to-one MAF-matched selections drawn from the 1000 Genomes GRCh38 high-coverage phased panel (2,504 unrelated samples, 2022 release):

  1. For each aging variant, we know its gnomAD allele frequency (from VEP).
  2. We convert AF → MAF = min(AF, 1-AF) and bin into one of five bins (<0.1%, 0.1-1%, 1-5%, 5-10%, 10-50%).
  3. We open the tabix-indexed 1000G VCF for that chromosome directly over HTTPS with pysam.VariantFile. Range requests fetch just the bytes for the target region (±50–1,000 kb, widening only when needed).
  4. For each record in the region, we read INFO/AF directly — no second lookup — and keep biallelic SNVs whose MAF bin matches the target.
  5. We exclude aging rsIDs and aging positions, then pick a random matching variant.
Result: each control is matched to one aging variant on MAF bin, chromosome, and genomic neighborhood. The binary task is balanced by construction (~5,000 vs ~5,000), so raw accuracy is no longer a trick. Smoke test (n=20): 100% match rate, per-bin matched distribution mirrors the aging bin distribution exactly.
Why not Ensembl REST? The first attempt used /overlap/region/variation + batch POST /variation/human. At scale it returned multi-megabyte JSON per query and timed out at 30–500 s per variant with 0% match rate. The tabix-over-HTTP path returns 3–4k records per query in ~1 s. Same science, 200× faster.

4. AlphaGenome inference step pretrained

Each variant gets scored by AlphaGenome exactly once and the result is cached to a Parquet file. AlphaGenome returns 19 AnnData objects containing predictions across 715 biosamples:

Output typeWhat it predicts
RNA_SEQGene expression change (per gene × biosample)
ATACChromatin accessibility
DNASEDNase hypersensitivity
CHIP_HISTONEHistone mark occupancy
CHIP_TFTranscription factor binding
+ active-track variants, CAGE, PROCAP, splicing, polyAAdditional regulatory outputs
from alphagenome.models import dna_client
from alphagenome.data import genome

model = dna_client.create(api_key)
scorers = dna_client.variant_scorers_lib.get_recommended_scorers(
    dna_client.Organism.HOMO_SAPIENS
)

variant = genome.Variant(chromosome=chrom, position=pos,
                         reference_bases=ref, alternate_bases=alt)
interval = genome.Interval(chromosome=chrom,
                           start=pos - 500_000, end=pos + 500_000)

results = model.score_variant(interval=interval, variant=variant,
                              variant_scorers=scorers)

A single variant produces about 26,500 prediction rows once flattened (715 biosamples × ~5 main output types × a small gene set for gene-level scorers). Per-variant compute is ~15–30 seconds on the hosted API.

5. Feature aggregation

26,500 raw scores per variant is more than a small downstream classifier can use. We aggregate to a 115-dimensional summary:

For each of 5 output types (RNA_SEQ, ATAC, DNASE, CHIP_HISTONE, CHIP_TF):

  • For each of 10 tissue categories (brain, heart, liver, blood, kidney, muscle, adipose, skin, colon, lung):
    • {ot}__{tissue}__max — max |score| (signed)
    • {ot}__{tissue}__mean_abs — mean |score|
  • {ot}__global__max — max |score| across all biosamples
  • {ot}__global__mean_abs — mean |score| across all biosamples
  • {ot}__global__std — std of raw scores

Total: 5 × (10 × 2 + 3) = 115 features.

Tissues are categorized by keyword matching on the biosample name (e.g. "cortex" → brain; "ventricle" → heart). The same exact aggregation runs at training time and at serving time so inference-time features are drop-in replacements for cached ones.

6. Splits & validation

This is the core of the v1 → v2 improvement. A naive random K-fold split of variants is not valid because variants cluster in genes and in linkage-disequilibrium blocks. If two variants from the same gene end up on either side of a train/test boundary, their AlphaGenome features are correlated and the model can memorize rather than generalize.

Gene-grouped splits

Every variant is assigned to a gene group (VEP gene_symbol when available, otherwise nearest-gene, otherwise a 100-kb position bucket). We use GroupShuffleSplit to carve a 20% held-out test set where no gene group crosses the boundary, then GroupKFold for 5-fold cross-validation on the training portion. 

from sklearn.model_selection import GroupShuffleSplit, GroupKFold

# 80/20 held-out by gene
splitter = GroupShuffleSplit(n_splits=1, test_size=0.2, random_state=42)
train_idx, test_idx = next(splitter.split(X, y, groups=gene_groups))

# 5-fold CV on training, still gene-grouped
cv = GroupKFold(n_splits=5)

No leakage in preprocessing

Imputer and scaler live inside the sklearn Pipeline, so they are refit on each training fold. In v1, StandardScaler was fit on combined aging+control data before CV — a minor leak for Random Forest, but enough that we codify the fix as a unit test (tests/test_no_leakage.py). 

from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler
from sklearn.ensemble import RandomForestClassifier

pipe = Pipeline([
    ("impute", SimpleImputer(strategy="constant", fill_value=0.0)),
    ("scale",  StandardScaler()),
    ("clf",    RandomForestClassifier(n_estimators=300, max_depth=10,
                                      class_weight="balanced",
                                      random_state=42)),
])

Leakage audit

We run both grouped and naive (random) 5-fold CV on the training set and report both AUCs. The delta is the LD/gene leakage that would have inflated metrics under v1's naive design.

7. Training the classifier

Binary task

Random Forest · 300 trees · max depth 10 · class_weight="balanced" (mostly cosmetic at 1:1) · trained on 80% of gene groups, tested on the held-out 20%.

Multi-class task

Same pipeline, 400 trees. Aging-only subset. Trait categories kept only if ≥30 variants. Majority-class and random (1/K) baselines reported alongside the test-set accuracy.

The v2.0 → v2.1 story. Two training runs shipped. The first (v2.0) used v1's 500 Ensembl-random controls and reported test AUC 0.838. Once v2.1 swapped in 5,054 MAF-matched controls from the 1000 Genomes high-coverage panel, test AUC dropped to 0.601. Same model, same pipeline, same code — the only change was making controls properly frequency-matched.
Read it honestly. v2.0's 0.838 AUC was mostly the classifier learning "is this variant common or rare?" — because aging variants are GWAS-discovered (preferentially common, mean MAF ~0.31) and v1 controls were Ensembl-random (93% unknown MAF, likely rare). With matched MAF distributions, the real signal is AUC 0.601 — modest, but above chance, and permutation- tested significant (p = 0.001). The leakage audit delta is +0.003 — essentially none — so the 0.838 → 0.601 drop is entirely the control- confound fix, not pipeline fixes.

Multi-class task — trait category (12 classes, aging-only)

Accuracy 0.396 0.362 majority 0.502 · random 0.083

Honest negative. Test balanced accuracy 0.096 is barely above the 0.083 random baseline. The classifier mostly predicts cardiovascular (50% of aging variants) and rarely gets other traits right. Trait-level regulatory signatures are not separable at this scale with this feature set — either the labels are too coarse, the feature aggregation is too lossy, or there simply isn't enough trait-specific signal in the AlphaGenome outputs we're using.

9. Inference pipeline (webapp)

The webapp is the point of the project — it demonstrates the whole pipeline end-to-end. Two modes:

Gallery (precomputed)

A curated set of well-known aging variants whose AlphaGenome scores are already cached. Click a card → instant prediction. Good for reviewers who don't want to wait.

Live inference

Enter any rsID or chr:pos:ref:alt. The backend:

  1. Resolves rsID → genomic coordinates via Ensembl REST
  2. Calls AlphaGenome (~30-60s per variant)
  3. Aggregates to 115 features
  4. Runs binary + multi-class classifiers
  5. Returns prediction + probability + tissue × output heatmap + top contributing features

10. Limitations

  • "Aging variant" is a heterogeneous label. Sources span GWAS trait associations, longevity cohorts, aging-gene regions, and curated literature. One label does not mean one biology.
  • Controls are frequency-matched, not trait-negative-verified. Required to be absent from the aging set and to match MAF/chromosome/neighborhood. Not guaranteed trait-free.
  • AlphaGenome predictions are not ground truth. They are model outputs. Any signal reflects AlphaGenome's priors.
  • Small decision layer, ~10k variants. Useful prior, not a diagnostic.
  • Missing populations. MAF is from 1000 Genomes / gnomAD, biased toward European ancestry.
